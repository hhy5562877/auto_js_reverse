# auto_js_reverse 里程碑

## M1: 稳定性基线

目标：让项目在缺少理想外部环境时仍能启动、抓取、归档与做基础分析。

交付项：

- Embedding 缺失时服务仍可启动
- 抓取目录改为会话级快照，避免覆盖历史数据
- 关键索引元数据保持一致
- 基础测试不依赖真实浏览器或外部 API
- 文档与代码行为一致

完成标准：

- 基础 MCP 服务可以在最小配置下启动
- `capture_current_page` 在未配置 Embedding 时仍可完成归档
- 新增至少一组纯逻辑回归测试

## M2: 分析能力增强

目标：让逆向流程从“能抓取”提升到“能高效定位”。

交付项：

- 提升 Source Map 发现与还原准确率
- 优化 `search_local_codebase` 的结果质量
- 加强 `capture_network_requests` 与 `hook_function` 的联动
- 增加对关键请求参数的结构化识别

完成标准：

- 能稳定定位登录签名、token、加密入口等常见场景
- 搜索和 Hook 结果能相互印证

## M3: 规模化归档与检索

目标：让项目能承载多域名、多会话、多版本的归档分析。

交付项：

- 优化大规模文件索引查询
- 引入分页、会话维度筛选、版本追踪
- 提供更稳健的文件与 chunk 关联关系

完成标准：

- 多次抓取同域名不会互相覆盖
- 查询性能不会随着归档数量线性失控

## M4: 产品化工作台

目标：把 `auto_js_reverse` 从一组工具演进为完整逆向工作台。

交付项：

- 域名级项目空间
- 任务编排与分析报告导出
- 规则库、模板库、案例库
- 从抓取到验证的闭环操作流

完成标准：

- 用户可以按项目管理站点、会话、结论和验证脚本
- 输出结果可复用、可分享、可复盘
