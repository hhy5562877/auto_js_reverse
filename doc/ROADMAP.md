# auto_js_reverse 发展计划

## 总体方向

`auto_js_reverse` 的核心价值，不只是“抓取网页 JS”，而是构建一个可复现、可验证、可沉淀的浏览器逆向分析系统。

项目发展分三条主线推进：

- 稳定性主线
- 分析能力主线
- 平台化主线

## 第一阶段：稳定性与工程化

阶段目标：

- 降低外部依赖对启动与基础功能的阻塞
- 保证抓取结果是可追踪的历史快照
- 让基础测试与最小验证路径可长期运行

重点任务：

- 继续清理强依赖初始化问题
- 优化本地依赖兼容性，特别是不同平台下的 LanceDB/uv 环境
- 将基础测试拆分为单元、集成、E2E 三层
- 统一日志、错误消息与故障提示

## 第二阶段：逆向分析能力增强

阶段目标：

- 提升定位加密逻辑、签名逻辑、调用链的效率
- 让搜索、Hook、网络监听形成组合拳

重点任务：

- 支持更稳定的 Source Map 发现逻辑
- 增强对混淆代码的切分与语义聚类
- 增加针对 `sign`、`token`、`encrypt`、`headers` 的专题分析模板
- 输出结构化分析结果，便于复用

## 第三阶段：归档与知识沉淀

阶段目标：

- 从“单次分析工具”变成“长期积累平台”

重点任务：

- 增加站点级、会话级、版本级元数据管理
- 支持分析报告和验证脚本导出
- 建立案例库、规则库、经验库

## 第四阶段：协作与产品化

阶段目标：

- 面向团队协作和复杂项目分析

重点任务：

- 增加任务队列和批量抓取
- 支持标准化分析工作流
- 为后续 Web UI 或桌面 UI 预留数据模型

## 当前优先级

短期内优先完成以下事项：

1. 修复剩余的基础稳定性问题
2. 降低测试和安装环境门槛
3. 提升索引查询与数据组织能力
4. 增强逆向分析闭环能力
